# Demo Video Script

Target length: **3 minutes 15 seconds to 3 minutes 45 seconds**

## 1. Problem and hook (0:00 - 0:20)

- Start on the homepage
- Say: "Most AI tutors still trap you in a text box. Rubik's Cubes are physical, visual, and live, so the interface should be too."

## 2. Show the routed product (0:20 - 0:40)

- Navigate from `/` to `/live`
- Briefly show `/labs/multiplayer`
- Mention the classic solver exists as a fallback demo mode

## 3. Live multimodal tutoring (0:40 - 2:00)

- Start the live session
- Show camera preview, transcript panel, and current instruction
- Speak a natural question like "What move should I do next?"
- Demonstrate that the tutor responds with live text and audio
- Interrupt it once to show barge-in handling
- Ask for a hint
- Show auto-solve preview or step-by-step coaching

## 4. Multiplayer lab (2:00 - 2:35)

- Open the multiplayer lab
- Create a room
- Explain that signaling happens through the backend and gameplay data goes peer-to-peer via WebRTC

## 5. Technical proof (2:35 - 3:05)

- Briefly show the Cloud Run health endpoint
- Show `/api/runtime`
- Mention the backend is on Google Cloud Run and uses the Google Gen AI SDK

## 6. Close (3:05 - 3:30)

- Return to the workspace
- Say: "AI Rubik's Tutor turns cube coaching into a real-time multimodal agent that can see, hear, speak, and respond in context."

## Recording Notes

- Use the physical cube if possible
- Keep one uninterrupted take of the live tutoring section
- Avoid mockups, slides-only segments, or static screenshots for the core feature proof
